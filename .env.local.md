# Environment Variables Setup

This file contains your Supabase credentials and site configuration.

## Important Notes

1. **Never commit `.env.local` to version control** - it contains sensitive credentials
2. The `.env.example` file shows the required variables without sensitive data
3. When deploying to Vercel, add these environment variables in your project settings

## Variables Explained

- `NEXT_PUBLIC_SUPABASE_URL`: Your Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Your Supabase anonymous/public key (safe to expose to clients)
- `NEXT_PUBLIC_SITE_URL`: Your site's URL (update to production URL when deploying)
- `NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL`: Development redirect URL for email verification

## Deployment

When deploying to Vercel:

1. Go to your project settings
2. Navigate to Environment Variables
3. Add all variables from `.env.local`
4. Update `NEXT_PUBLIC_SITE_URL` to your production domain (e.g., https://your-app.vercel.app)
